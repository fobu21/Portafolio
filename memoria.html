1. Tema o hilo conductor del portafolio

El hilo conductor del portafolio es la exploración de sistemas visuales que reaccionan, evolucionan o emergen a partir de reglas simples y de la interacción del usuario. Las obras no se plantean como imágenes finales, sino como procesos activos que se desarrollan en el tiempo.

El concepto de “sistema vivo” atraviesa todas las piezas: estructuras que responden a estímulos, crecen, se transforman o se reorganizan según condiciones internas o externas. Aunque cada obra utiliza técnicas distintas, todas comparten una intención común: hacer visible el comportamiento del sistema y no solo su resultado visual.

2. Qué se quiere expresar o investigar

Con este portafolio se investiga la relación entre control algorítmico y comportamiento emergente. A través del código, se busca entender cómo pequeñas reglas o inputs mínimos pueden generar resultados complejos y cambiantes.

También se explora el papel del espectador como parte activa del sistema. En lugar de un usuario pasivo, la interacción convierte al espectador en un elemento que activa i/o altera el comportamiento visual de cada obra.

Las piezas no buscan representar una narrativa cerrada, sino proponer una experiencia de observación del proceso.

3. Reflexión personal sobre el proceso creativo

El proceso creativo se ha basado en la experimentación constante. Muchas decisiones visuales no se tomaron desde el inicio, sino que surgieron al observar cómo el sistema se comportaba en ejecución y al iterar sobre los resultados.

Trabajar con p5.js ha reforzado la idea de que el error, el ajuste fino de parámetros y la iteración forman parte esencial del resultado final. En varios casos, pequeños cambios en el código produjeron transformaciones significativas en la estética, lo que llevó a replantear decisiones iniciales y a refinar el comportamiento de los sistemas.

4. Referentes e inspiración

  • Arte generativo: el proceso y la regla como núcleo de la obra.
  • Vida artificial: sistemas simples capaces de generar comportamientos complejos.
  • Autómatas: Conway y la complejidad a partir de reglas locales.
  • Fenómenos visuales: repetición, ritmo, crecimiento y transformación.

Estos referentes se entienden tanto como inspiración conceptual (emergencia, sistema, proceso) como técnica (reglas, iteración, simulación y recursión).

5. Decisiones visuales y técnicas relevantes

Visual

  • Estética abstracta y digital, centrada en el comportamiento más que en la representación.
  • Movimiento y transformación continua como base compositiva.
  • Uso de contraste, densidad y ritmo para enfatizar el cambio y la lectura del sistema.

Técnica

  • Interacción en tiempo real (input del usuario) como disparador o modulador.
  • Sistemas generativos con variación temporal y paramétrica.
  • Autómatas celulares (reglas locales y actualización por generaciones).
  • Recursión/fractal como estrategia para crear crecimiento y complejidad.

6. Cómo se relacionan entre sí las piezas

Las cuatro obras comparten:

  • Ejecución en tiempo real (el proceso sucede delante del espectador).
  • Reglas simples como base estructural.
  • Transformación continua (la obra no se fija en una única imagen).
  • Interacción como elemento clave para activar o modificar el sistema.

Aunque cada pieza explora un enfoque distinto (input directo, patrones generativos, autómatas, estructuras recursivas), todas dialogan como conjunto al presentar diferentes formas de comportamiento algorítmico. El portafolio funciona como una colección coherente porque cada obra aborda una dimensión del mismo interés: cómo el código puede producir sistemas visuales con un carácter orgánico y emergente.
